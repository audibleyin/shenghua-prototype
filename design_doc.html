一、系统架构设计

1.1 系统整体架构

生花AI创意平台采用分层架构设计，自顶向下分为用户层、应用层、支撑层、数据层，同时对接外部平台。各层职责明确，通过标准API接口进行数据交互。该架构参考美团/饿了么等成熟平台的分层设计理念，确保系统的高可用性、可扩展性和可维护性。

架构总览：
用户层：需求方（视频平台/品牌方/IP方/个人需求方）、创作者（内部学员/外部个人/工作室）、故事投稿者、资产上传者
应用层：首页（流量分发引擎）、接单大厅（交易撮合引擎）、创作广场（内容生态引擎）、资产库（数字资产引擎）、培训学院（人才孵化引擎）、人才信息库（人才发现引擎）
支撑层：等级体系（L1-L5五级认证）、交易保障（资金托管/仲裁/质控）、信用评价（双向评价+信用分）、消息通知（站内信/短信/推送）
数据层：用户中心、项目中心、内容中心、交易中心、培训中心
外部对接：抖音、红果短剧、快手、爱奇艺、B站/小红书

二、核心模块功能详细设计

2.1 首页（流量分发引擎）

首页是平台的流量调度中心，负责将不同类型的用户引导到最适合的功能模块。

区域内容功能逻辑
Hero Banner平台Slogan + 核心数据 + CTA按钮核心数据：12,680+注册创作者、3,450+完成项目、¥2,800万累计交易额、98.5%好评率
身份切换器头像旁身份标签，点击弹出下拉菜单已登录用户可一键切换身份端，导航项和默认页面随之变化

五级体系展示L1-L5等级卡片展示每级权益、可接项目类型、佣金比例
平台活动专区抖音/红果/快手/爱奇艺活动展示各平台最新创作活动和奖励
热门项目推荐最新/最热门项目卡片按用户等级展示可投标项目

2.2 接单大厅（交易撮合引擎）

平台核心交易模块，实现需求方与创作者的精准匹配和高效交易。

功能说明业务规则
项目浏览卡片式项目列表，展示预算/等级要求/截止日期按等级过滤：不同等级创作者看到不同项目池
多维筛选按类型/等级/预算/紧急程度筛选支持组合筛选，实时刷新结果
项目详情完整项目描述、交付标准、需求方信息弹窗展示，包含投标入口
投标/接单创作者提交投标申请需满足等级要求，附带个人简介和作品链接
交付管理分阶段验收、进度追踪按合同约定的里程碑节点验收
结算分账验收通过后自动释放资金扣除平台佣金后，创作者收到净额

等级匹配机制：L1新星可见预算≤¥2,000；L2新锐≤¥5,000；L3精英≤¥15,000；L4资深≤¥50,000；L5大师可见全部项目。

作品投稿创作者上传故事/剧本/样片/完整漫剧支持多种类型：故事投稿、漫剧样片、完整剧本

版权采购采购方在线选购作品版权支持议价，平台提供版权转让协议模板
互动社交浏览、点赞、评论、收藏、分享增加作品曝光，提升创作者影响力

2.6 等级体系（信用评价引擎）

等级名称可接项目佣金比例核心权益
L1新星入门项目（≤¥2,000）15%新手保护期、基础接单
L2新锐标准项目（≤¥5,000）12%更多项目可见、基础推荐
L3精英中型项目（≤¥15,000）10%品牌合作优先、团队功能
L4资深大型项目（≤¥50,000）8%高薪项目优先、专属顾问
L5大师全部项目（无上限）5%平台签约、独家项目、VIP资源

升降级规则：升级条件：完成指定数量项目+平均评分≥4.5星+通过对应等级考核+无重大违规记录。降级条件：连续3个月无活跃记录或评分降至3.5星以下或收到3次以上投诉。评级周期：每月1日系统自动结算，T+1公示。

2.7 社区（用户运营引擎）

创作者交流社区，共设8大板块：经验分享、作品互评、技术讨论、行业资讯、接单心得、学习打卡、问答求助、组队招募。核心功能包括发帖、互动（点赞/评论/收藏/分享）、热榜（综合浏览量/互动量/时效性计算）、活跃达人展示。

2.8 需求方服务

合作流程：发布项目 → 筛选创作者 → 托管资金 → 过程管理 → 验收结算。

发单表单字段：项目名称、项目类型（系列漫剧/品牌定制/单集短片/IP改编）、预算范围（¥500-2K / ¥2K-5K / ¥5K-15K / ¥15K-50K / ¥50K+）、等级要求、项目描述。

交易保障：资金托管（平台托管账户，验收后释放）、争议仲裁（48小时内裁决）、质量管控（分阶段验收）、信用评价（双向评价）。

2.9 创作者中心

创作者个人工作台，包含：数据总览（本月收入/进行中项目/待投标项目/资产收益）、我的订单、作品集、等级成长、收益明细、我的资产、培训记录、账号设置。

学费返还：仅线下VIP课程享有。每笔接单收入自动扣除20%-30%用于返还学费，累计不超过课程学费总额。多课程叠加时按报名顺序依次返还。

2.10 平台活动专区

聚合抖音、红果、快手、爱奇艺等主流平台的创作活动和激励计划。已对接：抖音（AI漫剧新力量大赛）、红果短剧（分账激励季）、快手（星芒AI漫剧专项）、爱奇艺（云腾计划AI漫剧赛道）。待开放：优酷、腾讯视频、微信视频号、B站、小红书。

2.11 人才信息库（人才发现引擎）

7大专业角色覆盖AI漫剧完整制作流程：导演、制作人、编剧、生成师、后期剪辑、美术、分镜师。生成师是AI特有的新角色，也是最大人群类别。

核心策略：先建库，后交易。前期吸引AI行业从业者+大学生提交作品审核，建立专业人才信息库；后期在信息库基础上撮合项目、对接需求方。

人才库不设单独登记入口，用户通过「成为创作者」→「我是老手」提交作品集，平台审核通过后直接进入人才库并获得等级，开通接单权限。审核不通过则无法入库。

三、数据库表结构设计

3.1 命名规范

表名：小写字母+下划线（user, creator_profile）。字段名：小写字母+下划线（user_id, created_at）。主键：统一 BIGINT 自增 id。外键：以 _id 结尾。索引：idx_ 前缀，唯一索引 uk_ 前缀。时间字段：统一 datetime。软删除：is_deleted（tinyint, 0=正常, 1=删除）。

3.2 用户中心域

1. 用户表（user） — 11字段：id(PK), phone(唯一), email, password_hash, nickname, avatar_url, status, register_source, last_login_at, is_deleted, created_at, updated_at。索引：uk_phone, idx_status, idx_nickname。

2. 用户身份表（user_identity） — 7字段：id, user_id(FK), identity_type(creator/client), is_active, profile_completed, created_at, updated_at。索引：uk_user_identity(user_id+identity_type), idx_user_active。

3. 用户认证表（user_auth） — 9字段：id, user_id(FK), real_name, id_card, auth_status(0未认证/1审核中/2已认证/3失败), auth_type, auth_time, created_at, updated_at。索引：uk_user_auth, idx_auth_status。

3.3 创作者体系域

4. 创作者档案表（creator_profile） — 19字段：id, user_id(FK), professional_role_id(FK), level_code, gender, city, experience_years, is_student, is_studio, bio, skills(JSON), representative_works, total_projects, total_income, avg_rating, is_verified, entry_path(training/portfolio), is_deleted, created_at, updated_at。索引：uk_creator_user, idx_level, idx_role, idx_city, idx_verified。

5. 创作者等级表（creator_level） — 预置L1-L5五级数据。字段：id, level_code(唯一), level_name, min_projects, min_rating, budget_limit, commission_rate, commission_rate_pro, commission_rate_elite, min_commission(1.00), monthly_order_limit, settlement_cycle, free_commission_orders, permissions(JSON), sort_order, is_active, created_at。

6. 创作者等级变更记录表（creator_level_log） — 7字段：id, user_id(FK), old_level, new_level, change_type(upgrade/downgrade/initial), reason, operator_id, created_at。

7. 专业角色表（professional_role） — 7大角色+细分方向。字段：id, role_code(唯一), role_name, parent_id(FK, 细分方向用), sort_order, is_active, created_at。

8. 创作者审核记录表（creator_review） — 13字段：id, user_id(FK), review_type(entry/level_up/appeal), status(0待审核/1通过/2不通过), quality_score, tech_score, experience_score, sustainability_score, total_score, assigned_level, review_comment, reviewer_id, reviewed_at, created_at。

3.4 人才信息库域

9. 人才信息表（talent_info） — 18字段：id, user_id(FK), professional_role_id(FK), display_name, gender, city, experience, is_student, is_studio, level_code, is_verified, is_available, skills(JSON), representative_works, stats_summary, is_in_talent_pool, is_deleted, created_at, updated_at。

10. 人才作品集表（talent_portfolio） — 9字段：id, talent_id(FK), title, description, file_url, file_type(video/image/pdf), sort_order, is_deleted, created_at, updated_at。

3.5 项目交易域

11. 项目表（project） — 21字段：id, client_id(FK), title, project_type(series/brand/single/ip), budget_min, budget_max, level_required, description, episode_count, duration_days, status(pending/open/in_progress/completed/cancelled/disputed), is_urgent, is_featured, is_sticky, deadline, bid_count, view_count, tags(JSON), is_deleted, created_at, updated_at。

12. 投标表（project_bid） — 8字段：id, project_id(FK), creator_id(FK), bid_amount, bid_message, status(pending/accepted/rejected/withdrawn), is_deleted, created_at, updated_at。唯一索引：uk_bid_unique(project_id+creator_id)。

13. 订单表（order） — 18字段：id, order_no(唯一), project_id(FK), client_id(FK), creator_id(FK), bid_id(FK), total_amount, commission_rate, commission_amount, creator_amount, escrow_amount, status(pending/escrowed/in_progress/delivered/reviewing/completed/cancelled/disputed), milestone_count, completed_milestones, is_free_commission, completed_at, is_deleted, created_at, updated_at。

14. 订单里程碑表（order_milestone） — 10字段：id, order_id(FK), milestone_name, sort_order, amount, status(pending/submitted/approved/rejected), delivery_note, delivery_url, approved_at, created_at, updated_at。

3.6 内容创作域

15. 作品表（work） — 18字段：id, creator_id(FK), title, work_type(story/sample/script/comic), description, cover_url, file_url, price, status(draft/published/sold), copyright_id(区块链版权登记ID), is_hot, is_new, view_count, like_count, comment_count, tags(JSON), is_deleted, created_at, updated_at。

16. 版权交易表（copyright_transaction） — 13字段：id, work_id(FK), buyer_id(FK), seller_id(FK), amount, commission_rate, commission_amount, platform_share, seller_share, status(pending/negotiating/completed/cancelled), contract_url, completed_at, is_deleted, created_at, updated_at。

3.7 数字资产域

18. 资产表（asset） — 14字段：id, uploader_id(FK), category_id(FK), title, description, thumbnail_url, file_url, price, is_exclusive, status(pending/approved/rejected/offline), review_comment, usage_count, is_deleted, created_at, updated_at。

五、关键业务流程

5.1 创作者入驻流程（双通道）

通道一：培训定级（零基础新手）
点击「成为创作者」→ 选择「我是新手」→ 进入培训学院 → 选课报名 → 在线学习 → 阶段考核 → 获得认证 → 自动定级 → 进入人才库 → 开通接单 → 学费返还。

通道二：作品定级（有经验老手）
点击「成为创作者」→ 选择「我是老手」→ 填写信息+上传作品（≥2个）→ 平台审核定级（1-3工作日）→ 审核通过→进入人才库+定级+接单。审核不通过→不入库，可补充作品后重新提交。

审核维度：作品质量40% · 技术能力25% · 项目经验20% · 持续能力15%

5.2 项目交易全流程

需求方发布项目→ 托管资金→ 系统按等级推送→ 创作者投标→ 需求方选标→ 分阶段交付→ 验收通过→ 释放资金
争议 → 平台仲裁 → 裁决执行

5.3 培训→接单闭环

报名课程 → 在线学习 → 阶段考核 → 获得等级 → 接单大厅 → 完成项目 → 学费返还。闭环效应：创作者赚到钱 → 口碑传播 → 吸引更多学员 → 培训收入增长 → 飞轮加速。

5.4 版权交易流程

创作者投稿 → 区块链确权 → 广场展示 → 采购方选中 → 议价签约 → 版权转让+分润。版权交易佣金8%（Business/Enterprise会员6%）。

5.5 佣金计算流程

公式：实际佣金 = 项目金额 × (等级佣金率 - 会员减免率)，最低1%。

示例：L3创作者标准佣金7%，购买Pro会员减免2%，实际佣金5%。L5创作者标准佣金3%，购买Elite会员减免3%，实际佣金1%（触达下限）。L1新手保护期：前5单免佣金，并赠送30天Pro会员免费体验。

6.4 双端身份体系设计

参照美团/饿了么的商家端与用户端模式，以及Boss直聘的招人版与求职版切换逻辑，平台采用「双端身份」设计：用户注册/登录时不绑定身份角色，注册后通过导航栏的切换按钮在「需求方」和「创作者」之间自由切换。

核心设计原则：① 一个账号，双重身份，无需注册两个账号。② 场景隔离，功能聚焦。③ 一键切换，无缝衔接。

维度需求方创作者
身份定位有AI漫剧制作需求的一方提供AI漫剧制作服务的一方
核心动作提需求：发布项目、筛选创作者、托管资金、验收交付成为创作者：提交作品审核、进入人才库、接单赚钱
导航可见项发项目、人才库、创作广场、资产库、社区接单大厅、创作者中心、培训学院、人才库、社区
首页CTA「我是需求方·发布项目」「成为创作者·接单赚钱」

6.5 创作者评级体系

双入口设计：培训定级入口（零基础新手，初始L1-L2）和作品定级入口（有经验从业者，初始L1-L4）。

动态升级机制：接单完成量40% + 用户反馈评分35% + 作品质量提升15% + 社区活跃度10%。升级条件：综合评分达到上一级标准 + 最近10个项目平均评分≥4.0。降级条件：连续3个项目评分低于3.5，或30天无接单活动。评级周期：每月1日系统自动结算。
